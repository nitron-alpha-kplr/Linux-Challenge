Solution Niveau 0
SSH, également connu sous le nom de Secure Shell ou Secure Socket Shell, est un protocole réseau qui offre aux utilisateurs, notamment aux administrateurs système, un moyen sécurisé d'accéder à un ordinateur via un réseau non sécurisé.

Secure Shell fournit une authentification forte par mot de passe et par clé publique, ainsi que des communications de données cryptées entre deux ordinateurs connectés sur un réseau ouvert, tel qu'Internet.

L'utilisation la plus élémentaire de SSH consiste à se connecter à un hôte distant pour une session de terminal. La syntaxe de cette commande est la suivante :

ssh UserName@SSHserver.example.com
Cette commande fera en sorte que le client tente de se connecter au serveur nommé server.example.com, en utilisant l'ID utilisateur UserName. 



Solution 
Dans notre cas, l'identifiant de l'utilisateur est bandit0 et le nom du serveur est bandit.labs.overthewire.org. Nous devons aussi spécifier le port 2220 en utilisant l'option "-p".

Ainsi, la commande pour passer au niveau suivant est la suivante :
ssh bandit0@bandit.labs.overthewire.org -p 2220
Tapez le mot de passe "bandit0",



Et voilà, vous vous êtes connectés.

Commande	Explication

ls -l	Lister les détails des fichiers
Comment cacher les fichiers et les répertoires dans Linux / Bash ?
Pour masquer un fichier ou un répertoire, il suffit d'ajouter un . au début du nom du fichier, comme suit :

Pour créer un fichier caché, exécutez touch .<insérer le nom du fichier>
Pour créer un répertoire caché, exécutez mkdir .<insérer le nom du répertoire>
La commande grep vs find
grep recherche une chaîne de caractères dans un fichier spécifié. Lorsqu'il trouve une correspondance, il imprime la ligne contenant le résultat.
find recherche des fichiers et des répertoires dans une hiérarchie de répertoires.

[Méthode 1] Utilisation de grep 

Notes sur la commande grep :
grep signifie recherche globale d'une expression régulière et impression de celle-ci (Global search for Regular Expression and Print out).
grep recherche les PATTERNS (expression régulière) dans chaque FILE (fichier).
La syntaxe de grep :

grep [OPTION] PATTERNS [FILE…]
Dans le cas présent,

PATTERNS : "millionth"
FILE : data.txt
Ainsi, la commande à exécuter est grep "millionth" data.txt

[Méthode 2] Utilisation de l'opérateur de pipe

Note sur l'opérateur de pipe :

L'opérateur pipe (| ou |&) de Linux prend la sortie standard d'une commande et la canalise vers l'entrée standard de la commande suivante.

Sous Linux, un pipe n'est pas une commande, mais un opérateur de contrôle.
La syntaxe de pipe (|) : 

Commande 1 | Commande 2 | Commande 3 | ... | Commande N
Dans le cas présent, la commande à exécuter est cat data.txt | grep "millionth".

Explication : 

Pour rechercher le mot 'millionth' dans le fichier data.txt, il faut d'abord lire son contenu. Pour lire son contenu, vous pouvez utiliser la commande cat en lui passant le chemin vers data.txt en entrée. Le contenu de data.txt est ensuite passé en entrée à la commande grep accompagné par 'millionth' comme deuxième entrée. Ainsi, vous avez recherché le mot 'millionth' dans le contenu de data.txt. C'est comme si vous avez exécuté grep 'millionth' [contenu_du_fichier_data_.txt].

Cela ne marche pas car d'après la description de uniq, il ne fait que "Filtrer les lignes adjacentes correspondantes". cat data.txt | sort | uniq
Cela ne marche pas car d'après la description de uniq, "Sans options, les lignes correspondantes sont fusionnées à la première occurrence". Ce n'est pas ce que nous voulons. Ainsi, nous ajoutons l'option -u pour "afficher uniquement les lignes uniques".

Informations complémentaires
Base64
Base64 est un groupe de schémas de codage binaire-texte qui représente les données binaires en séquences de 24 bits. 



Source : Base64 - Wikipedia

Codage
Sous Linux, les options -e ou --encode permettent de coder n'importe quelle donnée provenant d'un fichier ou de l'entrée standard. Elles sont passées par défaut à la commande base64 si vous ne précisez aucune option.
Pour coder, il y a quelques méthodes :

base64 nomfichier.txt ou base64 'string'
echo nomfichier.txt | base64 ou echo 'string' | base64
Cela codera les données du fichier nom_fichier.txt ou la chaîne de caractères 'string' et affichera les données codées en sortie dans le terminal.

Décodage
Les options -d ou --decode permettent de décoder les données codées par la base 64 à partir de n'importe quel fichier ou de l'entrée standard.
Pour décoder, il y a quelques méthodes :

base64 -d nom_fichier.txt ou base64 -d 'encodedString'
echo nomfichier.txt | base64 --decode ou echo 'encodedString' | base64 --decode
Cela décodera les données dans nomfichier.txt ou la chaîne codée 'encodedString' et affichera les données décodées (texte original) comme sortie dans le terminal.

Note
Encoding ≠ Encrypting
Codage ≠ Chiffrement
Pour plus de détails sur leur différence :
Différence entre codage et chiffrement
Codage vs. chiffrement vs. hachage vs. obscurcissement

Exemple :

>../folderName$ cat index.txt

>../folderName$ cat index.txt | tr "a-z" "A-Z"
HELLO WORLD%
Dans le cas présent, pour décoder le contenu du fichier data.txt, la commande à utiliser est tr "A-Za-z" "N-ZA-Mn-za-m". Cela signifie que pour :

Les majuscules : Mapper de A-Z à N-ZA-M
Minuscules : Mapper de a-z à n-za-m
Ainsi, l'une des commandes suivantes affichera le contenu décodé du fichier data.txt :

cat data.txt | tr "A-Za-z" "N-ZA-Mn-za-m" 
cat data.txt | tr "a-zA-Z" "n-za-mN-ZA-M"
cat data.txt | tr "[a-zA-Z]" "n-za-mN-ZA-M"
cat data.txt | tr "n-za-mN-ZA-M" "a-zA-Z"

Informations complémentaires
Jusqu'à présent, vous avez utilisé l'authentification SSH par mot de passe pour vous connecter à un serveur distant : 
ssh <Nom d'utilisateur>@<Nom d'hôte> -p <Numéro de port> et entrer le mot de passe.
Cependant, comme mentionné dans la page SSH/OpenSSH/Keys, l'authentification basée sur une clé est plus sécurisée que l'authentification par mot de passe.
En bref, l'authentification par clé utilise 2 clés, une clé "publique" que tout le monde est autorisé à voir, et une autre clé "privée" que seul le propriétaire est autorisé à voir. Pour communiquer en toute sécurité à l'aide de l'authentification par clé, il faut créer une paire de clés, stocker en toute sécurité la clé privée sur l'ordinateur à partir duquel on veut se connecter et stocker la clé publique sur l'ordinateur auquel on veut se connecter.

Qu'est-ce que Setuid ?
Setuid ( Set User Identity ) est un paramètre de fichier Linux qui permet à un utilisateur d'exécuter un exécutable avec les autorisations de système de fichiers du propriétaire de l'exécutable et de modifier le comportement dans les répertoires. Ceci est principalement utilisé pour élever les privilèges de l'utilisateur actuel.

Setuid est un moyen pour les utilisateurs d'exécuter un exécutable avec les permissions de l'utilisateur qui possède le fichier. Par exemple, si vous souhaitez qu'un utilisateur soit en mesure d'effectuer une tâche spécifique nécessitant les privilèges de root/superuser, mais que vous ne souhaitez pas lui donner l'accès sudo ou root.

Pour trouver le mot de passe du niveau 21
[Étape 1] : Connectez-vous et connectez-vous au compte avec le nom d'utilisateur et le mot de passe indiqués ci-dessus.



[Étape 2] : Exécuter ls -l dans le répertoire de travail actuel pour identifier le fichier avec un paramètre d'autorisation de fichier « setuid ».

Note : 
Lorsque vous vous connectez pour la première fois, votre répertoire de travail actuel est votre répertoire personnel. Votre répertoire personnel porte le même nom que votre nom d'utilisateur. Par exemple, ee91ab, et c'est là que vos fichiers et sous-répertoires personnels sont enregistrés.
 bandit20@bandit:~$ ls -l
total 12
-rwsr-x--- 1 bandit21 bandit20 12088 7 mai 20:14 suconnect
Comme mentionné dans le niveau précédent , dans l'image ci-dessus, le fichier suconnect a un bit s situé dans les classes d'autorisation "Utilisateur". Par conséquent, c'est le fichier que nous trouvons avec le jeu d'autorisations binaires setuid .



De plus, le propriétaire du fichier suconnect est bandit21. La surbrillance rouge signifie que ce fichier a des autorisations élevées. Par conséquent, toutes les commandes exécutées avec s'exécuteront en tant qu'utilisateur au lieu de .bandit21./suconnect bandit21 bandit20



[Étape 3] : Exécutez ./suconnect la commande pour comprendre le programme et comment l'utiliser pour récupérerle mot de passe de bandit21.

bandit20@bandit:~$ ./suconnect
Usage: ./suconnect <portnumber>
This program will connect to the given port on localhost using TCP. If it receives the correct password from the other side, the next password is transmitted back.


"from the other side" est un indice indiquant que ce niveau nécessite que 2 terminaux fonctionnent en conjonction entre eux :

Le 1er Terminal démarrera un écouteur netcat sur un numéro de port choisi &
Le 2ème Terminal s'y connectera à l'aide ./suconnect <Port Number> de la commande.


Note : 
Vous devez choisir un numéro de port non utilisé. Vous pouvez vérifier les numéros de ports utilisés grâce à la commande nmap localhost. Puis, choisissez n'importe quel numéro de port non répertorié dans la sortie. Dans ce cas, le port choisi sera 6000.


Une fois configuré, dans le 1er Terminal , entrez le mot de passe de . Le mot de passe sera envoyé via netcat pour lecture. Une fois qu'il a reçu et validé le mot de passe saisi, il répondra avec le mot de passe de bandit21



[Étape 4] : Pour démarrer un écouteur netcat, exécutez nc -lp 6000 .

-l : Pour spécifier que nc doit écouter une connexion entrante plutôt que d'initier  une connexion à un hôte distant.
-p <Port> : Spécifie le numéro de port que nc doit utiliser.


Rappel :
netcat est un simple utilitaire Unix qui lit et écrit des données sur les connexions réseau, en utilisant le protocole TCP ou UDP.


[Étape 5] : Une fois que netcat commence à écouter sur le port 6000, ouvrez un 2e terminal. Connectez-vous et connectez-vous au même compte que dans [# Étape 1] . Ensuite, exécutez ./suconnect 6000 pour vous connecter au port 6000 sur localhost en utilisant TCP.





[Étape 6] : De retour au 1er Terminal , saisissez le mot de passe de . Si le mot de passe est correctement entré, il affichera le mot de passe de bandit21

Qu’est-ce que Cron, CronJob et Crontab ?
Cron
Cron est un un planificateur de tâches basé sur le temps
dans les systèmes d'exploitation informatiques Unix ou de
type Unix.
Il planifie une commande ou un script sur votre serveur
pour qu'il s'exécute automatiquement en arrière-plan, à
un moment désigné.
Qu’est-ce que Cron, CronJob et Crontab ?
CronJob (Tâche Cron)
Une tâche Cron est la tâche planifiée elle-même.
Exemple de tâche :
Supprimer les fichiers temporaires chaque semaine afin de conserver
votre espace disque.
Les tâches cron peuvent être configurées pour s'exécuter à
la minute, à l'heure, au jour de la semaine, au jour du
mois, au mois ou à une combinaison de ces éléments.
